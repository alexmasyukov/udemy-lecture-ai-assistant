# Udemy Lecture AI

Chrome-расширение для работы с лекциями Udemy через локальную LLM (LM Studio или любой OpenAI-совместимый endpoint). Вытягивает транскрипт текущей лекции, даёт по нему саммари и отвечает на вопросы — или просто работает как обычный чат, когда транскрипт не нужен.

## Что умеет

- Тянет транскрипт через официальный Udemy API (`/api-2.0/.../lectures/.../?fields[asset]=captions`) — не зависит от того, открыта ли панель транскрипта на странице.
- По умолчанию берёт ручные английские субтитры; если их нет — любые английские, потом manual, потом первые доступные.
- Автоматически перезагружает транскрипт при SPA-переходе между лекциями (через `chrome.webNavigation.onHistoryStateUpdated`).
- Если content script ещё не был инжектнут в таб — панель инжектит его через `chrome.scripting.executeScript`.
- Таймкоды подставляются в system prompt — модель может ссылаться на `[mm:ss]`.
- Стриминг ответов: слова появляются по мере генерации (SSE, как в ChatGPT).
- Ответы рендерятся в Markdown через `marked.js` (GFM, таблицы, код, списки).
- Кнопка **Summary** — генерация саммари; модель сама выбирает структуру и объём.
- Свободный чат работает и без загруженного транскрипта.
- Тумблер **Lecture context only** (включён по умолчанию) — строгий режим, только по транскрипту. Отжал — можно задавать любые вопросы, модель использует общие знания.
- Настраиваемые размеры шрифтов UI и чата.
- Локальная модель через OpenAI-совместимый API (по умолчанию LM Studio на `http://127.0.0.1:1234/v1`).
- Резервный DOM-парсинг панели транскрипта, если API недоступен.

## Установка

1. Склонируй папку `/Users/alex/chrome_plugins/udemy`.
2. Открой `chrome://extensions/`, включи **Developer mode** (правый верхний угол).
3. Нажми **Load unpacked** и выбери папку с расширением.
4. Запусти [LM Studio](https://lmstudio.ai) и включи локальный сервер (вкладка **Developer → Start Server**). По умолчанию он слушает `127.0.0.1:1234`.
5. Загрузи любую модель (по умолчанию в расширении прописана `gemma-4-e4b-it`).

## Использование

1. Открой лекцию Udemy: `https://www.udemy.com/course/.../learn/lecture/...`.
2. Кликни по иконке расширения в панели Chrome — справа откроется side panel.
3. Нажми **Reload transcript** — расширение подтянет все cues через API (при переходе на другую лекцию это произойдёт автоматически).
4. **Summary** — автоматическое саммари. Или задай любой вопрос в поле внизу.
5. **Lecture context only** (верхняя панель) — включён: ответы строго по транскрипту. Отжат: свободный чат с общими знаниями.

Настройки (⚙ в правом верхнем углу):

- **Base URL** — адрес OpenAI-совместимого API.
- **Model** — выпадашка моделей, подтягивается из `/v1/models`. Кнопка ↻ обновляет список.
- **Temperature** — сэмплинг (по умолчанию `0.3`).
- **UI font size** — размер шрифта интерфейса (по умолчанию `13px`).
- **Chat font size** — размер шрифта чата (по умолчанию `16px`).

```

├── sidepanel.css            # тёмная тема, CSS-переменные для шрифтов
├── sidepanel.js             # логика UI, стриминг чата, история, настройки
└── vendor/
    └── marked.min.js        # markdown → HTML (MIT)
```

## Как достаётся транскрипт

1. На странице `/course/{slug}/learn/lecture/{id}` есть элемент с атрибутом `data-module-args`, где JSON содержит `courseId`.
2. `lectureId` берётся из URL (`/lecture/{id}`) — так он всегда свежий при SPA-переходе. `initialCurriculumItemId` из `data-module-args` используется только как fallback.
3. Content script дёргает:
   ```
   GET /api-2.0/users/me/subscribed-courses/{courseId}/lectures/{lectureId}/?fields[asset]=captions
   ```
   (куки юзера уходят автоматически через `credentials: 'include'`)
4. В ответе массив `captions` с подписанными VTT URL по всем доступным языкам.
5. Выбирается manual English → скачивается VTT → парсится в `{start, end, text}[]`.
6. Текст с таймкодами уходит в system prompt LLM.

## Как работает чат

Запрос к LLM (`/v1/chat/completions` с `stream: true`) уходит **напрямую** из side panel (host_permissions для 127.0.0.1), минуя service worker. Ответ читается через `ReadableStream.getReader()`, SSE-строки `data: {...}` парсятся, `choices[0].delta.content` аккумулируется и на каждый chunk перерисовывается bubble через `marked.parse()`.

Каждый запрос содержит:

- **system**: инструкция + (опционально) транскрипт с таймкодами.
- **history**: предыдущие user/assistant пары текущей сессии.
- **user**: новое сообщение.

История хранится в памяти side panel и **сохраняется** при переходе между лекциями — обновляется только транскрипт в system prompt. Сбрасывается при закрытии панели.

## Известные ограничения

- Side panel не переживает закрытие — история чата теряется.
- Длинные лекции (>80k токенов) могут не влезть в контекст модели — чанкинг не реализован.
- Content script инжектится автоматически при SPA-навигациях, но при первой установке расширения уже открытые табы требуют перезагрузки — это стандартное поведение Chrome.
