# Udemy Lecture AI

Chrome-расширение, которое вытаскивает транскрипт текущей лекции Udemy и общается с локальной LLM (LM Studio или любой OpenAI-совместимый endpoint) — можно попросить саммари или задать вопросы по содержанию.

## Что умеет

- Достаёт транскрипт через официальный Udemy API (`/api-2.0/.../lectures/.../?fields[asset]=captions`) — не зависит от открытой панели на странице.
- Выбор языка субтитров (dropdown всех доступных дорожек, с приоритетом manual English).
- Таймкоды подставляются в промпт — модель может ссылаться на `[mm:ss]`.
- Саммари одной кнопкой + свободный чат по транскрипту.
- Локальная модель через OpenAI-совместимый API (по умолчанию LM Studio на `http://127.0.0.1:1234/v1`).
- Резервный DOM-парсинг панели транскрипта, если API недоступен.

## Установка

1. Склонируй или скачай папку `/Users/alex/chrome_plugins/udemy`.
2. Открой `chrome://extensions/`, включи **Developer mode** (правый верхний угол).
3. Нажми **Load unpacked** и выбери папку с расширением.
4. Запусти [LM Studio](https://lmstudio.ai) и включи локальный сервер (вкладка **Developer** → **Start Server**). По умолчанию он слушает `127.0.0.1:1234`.
5. Загрузи любую модель (по умолчанию расширение использует `gemma-4-e4b-it`).

## Использование

1. Открой лекцию Udemy: `https://www.udemy.com/course/.../learn/lecture/...`.
2. Кликни по иконке расширения в панели Chrome — справа откроется side panel.
3. Нажми **Load transcript** — расширение подтянет все cues через API.
4. **Summarize** — автоматическое саммари. Или задай вопрос в поле внизу.
5. Язык субтитров меняется в dropdown (при смене транскрипт перезагружается автоматически).

Настройки (шестерёнка сверху справа):

- **Base URL** — адрес OpenAI-совместимого API (по умолчанию `http://127.0.0.1:1234/v1`).
- **Model** — выпадашка моделей, подтягивается из `/v1/models`. Кнопка ↻ обновляет список.
- **Temperature** — сэмплинг (по умолчанию `0.3`).

## Структура

```
.
├── manifest.json      # MV3 manifest, content script + side panel + SW
├── background.js      # service worker: прокси к LLM, настройки
├── content.js         # вытягивает courseId/lectureId из data-module-args,
│                      # дёргает captions API, парсит VTT
├── sidepanel.html     # UI боковой панели
├── sidepanel.css      # тёмная тема
└── sidepanel.js       # логика UI, чат, история, выбор модели
```

## Как оно достаёт транскрипт

1. На странице `/course/{slug}/learn/lecture/{id}` есть элемент с атрибутом `data-module-args`, где JSON содержит `courseId` и `initialCurriculumItemId`.
2. Content script парсит этот JSON и дёргает:
   ```
   GET /api-2.0/users/me/subscribed-courses/{courseId}/lectures/{lectureId}/?fields[asset]=captions
   ```
   (куки юзера уходят автоматически через `credentials: 'include'`)
3. В ответе массив `captions` с подписанными VTT URL по всем доступным языкам.
4. Выбирается `manual en_US` → скачивается VTT → парсится в `{start, end, text}[]`.
5. Текст с таймкодами уходит в system prompt LLM.

## Логика чата

Каждый запрос к LLM содержит:

- **system**: инструкция «отвечай только на русском, только по транскрипту» + сам транскрипт с таймкодами.
- **history**: предыдущие user/assistant пары текущей сессии.
- **user**: новое сообщение.

История хранится в памяти side panel (сбрасывается при новой загрузке транскрипта).

## Известные ограничения

- Side panel не переживает закрытие — история чата теряется.
- Длинные лекции (>80k токенов) могут не влезть в контекст модели — чанкинг пока не реализован.
- Требуется свежий запуск страницы после установки расширения (content script не инжектится в уже открытые вкладки — это стандартное поведение Chrome).
